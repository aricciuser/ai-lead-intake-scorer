# AI Lead Intake Scorer

A simple browser app where you paste in a sales lead, click a button, and get
back a fit score, priority label, recommended next step, and a draft follow up
message — all generated by Claude.

This is your capstone tool for the course. The goal is to learn the workflow:
**form → prompt → AI response → test → commit → deploy → improve.**

## Quick start (15 minutes)

These steps work on Mac, Windows, and Linux. Where commands differ, both
versions are shown.

### 1. Install Node.js

You need Node 18.18 or newer. Open a terminal and check with:

```bash
node --version
```

If you do not have it (or it is too old), install from
[nodejs.org](https://nodejs.org). Pick the **LTS** version.

> **Windows users:** Use **PowerShell** for the commands in this README.
> Right click the Start menu and pick **Terminal** or **Windows PowerShell**.

### 2. Get the code

Clone this repo to your computer:

```bash
git clone https://github.com/aricciuser/ai-lead-intake-scorer.git
cd ai-lead-intake-scorer
```

> If you do not have git, install it from [git-scm.com](https://git-scm.com).
> All terminal commands below should be run from inside the
> `ai-lead-intake-scorer` folder.

### 3. Install dependencies

```bash
npm install
```

This downloads everything the project needs. It takes 30-60 seconds.

### 4. Get a Claude API key

1. Go to [console.anthropic.com](https://console.anthropic.com)
2. Sign up or log in
3. Click **Settings → API Keys → Create Key**
4. Copy the key (starts with `sk-ant-...`)

### 5. Add your key

Make a copy of `.env.example` named `.env.local`.

**Mac / Linux** (Terminal):
```bash
cp .env.example .env.local
```

**Windows** (PowerShell):
```powershell
Copy-Item .env.example .env.local
```

**Windows** (Command Prompt):
```cmd
copy .env.example .env.local
```

Open `.env.local` in your editor and paste your key after `ANTHROPIC_API_KEY=`.

> The `.env.local` file is already in `.gitignore`. Never commit it.

### 6. Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Fill in the form with a
test lead and click **Score this lead**. You should see a score and follow up
message in a few seconds.

## Customize for your business

Open `lib/scoringPrompt.ts`. The top of the file has a **CUSTOMIZE THIS FILE**
banner. Edit these values:

1. **`COMPANY`** — your company name and one-line description.
2. **`TARGET_CUSTOMER`** — who you sell to.
3. **`BUYING_SIGNALS`** — what tells you a lead is a good fit.
4. **`POOR_FIT_SIGNALS`** — what tells you a lead is a bad fit.
5. **`PRIORITY_BANDS`** — score ranges and what they mean.
6. **`FOLLOW_UP_TONE`** — the voice of the draft follow up message.

Save the file. The dev server reloads automatically. Score a test lead — the
output should reflect your changes.

### Other things to customize

| File | What to change |
|------|----------------|
| `components/LeadForm.tsx` | Add or remove form fields |
| `app/page.tsx` | Page headline and subhead |
| `components/LeadForm.tsx` | Submit button text (search "Score this lead") |
| `tailwind.config.ts` | Brand colors (later — get the basics working first) |

If you add or rename a form field, also update the `Lead` type and
`buildUserPrompt()` in `lib/scoringPrompt.ts` so the AI sees the new field.

## Test it in the browser

1. Run `npm run dev`
2. Open [http://localhost:3000](http://localhost:3000)
3. Try **three test leads**:
   - One that is clearly a great fit (should score high)
   - One that is clearly a bad fit (should score low)
   - One in the middle (should score in the middle)
4. Read the follow up messages out loud. Do they sound like you?
5. If the scores or messages feel off, edit `lib/scoringPrompt.ts` and try again.

This is the loop: **change the prompt, score a lead, read the result, change again.**

## Push your version to your own GitHub repo

You already cloned this repo, so git is set up. To save your customized
version under your own account:

1. Go to [github.com](https://github.com) and create a **new empty repo**.
   Pick any name you want (e.g. `my-lead-scorer`). Do **not** initialize it
   with a README.
2. Point this folder at your new repo and push:

```bash
git remote set-url origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
git add .
git commit -m "Customize lead scorer for my business"
git push -u origin main
```

Replace `YOUR_USERNAME` and `YOUR_REPO_NAME` with your actual values.

> If `git push` asks for a password, GitHub no longer accepts your account
> password here. Generate a **Personal Access Token** at
> [github.com/settings/tokens](https://github.com/settings/tokens) and use
> that as the password.

## Deploy to Vercel

1. Go to [vercel.com](https://vercel.com) and sign in with GitHub.
2. Click **Add New → Project** and pick your repo.
3. On the configure screen, expand **Environment Variables** and add:
   - **Name:** `ANTHROPIC_API_KEY`
   - **Value:** your key (the same `sk-ant-...` from `.env.local`)
4. Click **Deploy**.
5. Wait one to two minutes. You will get a public URL like
   `your-project.vercel.app`.

Open the URL on your phone. Score a lead. That is your tool, live on the
internet.

Every time you `git push` to GitHub, Vercel redeploys automatically.

## Troubleshooting

**"Server is missing ANTHROPIC_API_KEY"**
→ Make sure you copied `.env.example` to `.env.local` and pasted your real key.
Restart `npm run dev` after editing the file.

**"AI error: 401 authentication_error"**
→ Your key is wrong, copied with extra spaces, or got revoked. Get a new one
from [console.anthropic.com](https://console.anthropic.com).

**The score feels wrong**
→ Edit `lib/scoringPrompt.ts`. The AI only knows what you tell it.

**Vercel deploy fails with "ANTHROPIC_API_KEY is missing"**
→ You forgot to add the env variable in Vercel. Go to your project →
**Settings → Environment Variables** and add it. Then redeploy.

**Page is blank**
→ Open the browser dev tools (F12) and check the Console tab for errors.
