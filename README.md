# AI Lead Intake Scorer

A simple browser app where you paste in a sales lead, click a button, and get
back a fit score, priority label, recommended next step, and a draft follow up
message — all generated by an AI model (ChatGPT or Claude — your choice).

This is your capstone tool for the course. The goal is to learn the workflow:
**form → prompt → AI response → test → commit → deploy → improve.**

## Quick start (15 minutes)

These steps work on Mac, Windows, and Linux. Where commands differ, both
versions are shown.

### 1. Install Node.js

You need Node 18.18 or newer. Open a terminal and check with:

```bash
node --version
```

If you do not have it (or it is too old), install from
[nodejs.org](https://nodejs.org). Pick the **LTS** version.

> **Windows users:** Use **PowerShell** for the commands in this README.
> Right click the Start menu and pick **Terminal** or **Windows PowerShell**.

### 2. Get the code

Clone this repo to your computer:

```bash
git clone https://github.com/aricciuser/ai-lead-intake-scorer.git
cd ai-lead-intake-scorer
```

> If you do not have git, install it from [git-scm.com](https://git-scm.com).
> All terminal commands below should be run from inside the
> `ai-lead-intake-scorer` folder.

### 3. Install dependencies

```bash
npm install
```

This downloads everything the project needs. It takes 30-60 seconds.

### 4. Get an AI API key

This project supports **two AI providers**. You only need a key for one:

| Provider | Default? | Get a key at | Key starts with |
|----------|----------|--------------|-----------------|
| OpenAI / ChatGPT | ✅ Yes | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | `sk-...` |
| Anthropic / Claude | No | [console.anthropic.com](https://console.anthropic.com) → Settings → API Keys | `sk-ant-...` |

**You don't have to choose just one.** The app has a **ChatGPT / Claude
toggle** at the top of the form, so you can switch between providers per
request. If you only have one key, that's the only side that will work —
the other will return an error until you add its key.

> Both providers have free signup credits. Pick whichever you prefer — the
> app works the same either way.

### 5. Add your key

Make a copy of `.env.example` named `.env.local`.

**Mac / Linux** (Terminal):
```bash
cp .env.example .env.local
```

**Windows** (PowerShell):
```powershell
Copy-Item .env.example .env.local
```

**Windows** (Command Prompt):
```cmd
copy .env.example .env.local
```

Open `.env.local` in your editor and paste your key(s) on the matching line:

- **OpenAI / ChatGPT:** paste after `OPENAI_API_KEY=`
- **Anthropic / Claude:** paste after `ANTHROPIC_API_KEY=`

You can fill in just one or both. Whichever ones you fill in are the
provider buttons that will work in the form's toggle. If you only paste an
OpenAI key, the **Claude** button will return an error until you add an
Anthropic key.

> The `.env.local` file is already in `.gitignore`. Never commit it.

### 6. Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). At the top of the
form, pick **ChatGPT** or **Claude**. Fill in the rest, then click
**Score this lead**. You should see a score and a follow up message in a
few seconds.

The result screen shows which model scored the lead so you can compare.
Score the same lead with both to see how they differ.

## Customize for your business

Open `lib/scoringPrompt.ts`. The top of the file has a **CUSTOMIZE THIS FILE**
banner. Edit these values:

1. **`COMPANY`** — your company name and one-line description.
2. **`TARGET_CUSTOMER`** — who you sell to.
3. **`BUYING_SIGNALS`** — what tells you a lead is a good fit.
4. **`POOR_FIT_SIGNALS`** — what tells you a lead is a bad fit.
5. **`PRIORITY_BANDS`** — score ranges and what they mean.
6. **`FOLLOW_UP_TONE`** — the voice of the draft follow up message.

Save the file. The dev server reloads automatically. Score a test lead — the
output should reflect your changes.

### Other things to customize

| File | What to change |
|------|----------------|
| `components/LeadForm.tsx` | Add or remove form fields |
| `app/page.tsx` | Page headline and subhead |
| `components/LeadForm.tsx` | Submit button text (search "Score this lead") |
| `tailwind.config.ts` | Brand colors (later — get the basics working first) |

If you add or rename a form field, also update the `Lead` type and
`buildUserPrompt()` in `lib/scoringPrompt.ts` so the AI sees the new field.

## Test it in the browser

1. Run `npm run dev`
2. Open [http://localhost:3000](http://localhost:3000)
3. Try **three test leads**:
   - One that is clearly a great fit (should score high)
   - One that is clearly a bad fit (should score low)
   - One in the middle (should score in the middle)
4. Read the follow up messages out loud. Do they sound like you?
5. If the scores or messages feel off, edit `lib/scoringPrompt.ts` and try again.

This is the loop: **change the prompt, score a lead, read the result, change again.**

Replace `YOUR_USERNAME` and `YOUR_REPO_NAME` with your actual values.

## Deploy to Vercel

### First-time deploy

1. Go to [vercel.com](https://vercel.com) and sign in with GitHub.
2. Click **Add New → Project** and pick your repo.
3. On the configure screen, expand **Environment Variables** and add the
   key (see the section below for the exact steps and values).
4. Click **Deploy**.
5. Wait one to two minutes. You will get a public URL like
   `your-project.vercel.app`.

Open the URL on your phone. Score a lead. That is your tool, live on the
internet.

Every time you `git push` to GitHub, Vercel redeploys automatically.

### Add your AI API key(s) to Vercel

The deployed site will load without keys, but every scoring request will
fail until at least one provider's key is set. Add the key for each
provider you want to use — students typically add both so they can
compare ChatGPT and Claude in the live demo.

**Step 1 — In Vercel, go to your project → Settings → Environment Variables.**

**Step 2 — Add the API key(s).** Click **Add New** for each one:

| Provider | Variable name | Value |
|----------|---------------|-------|
| OpenAI / ChatGPT | `OPENAI_API_KEY` | your `sk-...` from [platform.openai.com/api-keys](https://platform.openai.com/api-keys) |
| Anthropic / Claude | `ANTHROPIC_API_KEY` | your `sk-ant-...` from [console.anthropic.com](https://console.anthropic.com) |

For each variable, check **Production**, **Preview**, AND **Development**
so it works on every branch.

**Step 3 — Save.**

**Step 4 — Trigger a redeploy.** Vercel does **not** auto-redeploy when you
change env vars:
- Go to the **Deployments** tab
- Find the most recent deployment, click the **⋯** menu → **Redeploy**
- Or push any commit (`git commit --allow-empty -m "trigger redeploy"`)

**Step 5 — Wait for the new build (~1-2 minutes), then test the live URL.**
Pick a provider in the form's toggle and score a lead. If you only added
one key, the other side of the toggle will return an error until you add
its key.

> **Security note:** Your API key never appears in the browser. The app keeps
> it server-side in `app/api/score-lead/route.ts`. Anyone with your live URL
> can score leads (which costs you a few cents per call), so don't share the
> URL publicly until you decide that's fine.

> **Rotating a key:** If you ever need to change a key (revoked, leaked,
> moved to a new account), edit the existing variable in Vercel instead of
> adding a new one, then redeploy.

## Troubleshooting

**"Missing OPENAI_API_KEY" or "Missing ANTHROPIC_API_KEY"**
→ Make sure you copied `.env.example` to `.env.local` and pasted your real
key for whichever provider you're using. Restart `npm run dev` after
editing the file (the dev server only reads `.env.local` at startup).

**"401" or "authentication" error**
→ Your key is wrong, copied with extra spaces, or got revoked. Get a new
one from [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
or [console.anthropic.com](https://console.anthropic.com).

**You want to switch providers**
→ Use the **ChatGPT / Claude** toggle at the top of the form in your
browser. Make sure the matching API key is in `.env.local` (locally) or
set in Vercel's env vars (production).

**The score feels wrong**
→ Edit `lib/scoringPrompt.ts`. The AI only knows what you tell it.

**Live Vercel site says "Missing OPENAI_API_KEY" (or ANTHROPIC_API_KEY)**
→ The build deployed but the env variable is not set on Vercel. Follow
**Add your AI API key to Vercel** above. Remember to redeploy after
saving — Vercel does not auto-redeploy when env vars change.

**Page is blank**
→ Open the browser dev tools (F12) and check the Console tab for errors.
